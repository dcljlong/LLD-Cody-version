Project overview

We are developing two separate apps:

1. FitoutOS (data engine / planning system)

Purpose:

Quotes
Programmes
Resource analysis
Task codes
Crew loading
Scope parsing
Programme parsing
Scheduling logic
Reports

FitoutOS is the source of truth for construction data.

Stack:

FastAPI backend
MongoDB
React frontend
Runs locally on port 8010
2. LLD Cody Version (site operations app)

Purpose:

Daily site use
Walkaround capture
Action items
Diary
Gates / risks
Dashboard
Project visibility

This app is for on-site use on phone / tablet / laptop.

It must stay lightweight.

LLD must NOT contain heavy planning logic.

LLD will later pull data from FitoutOS via API.

Stack:

FastAPI backend
MongoDB
React frontend
Runs locally on port 8011 (important)
IMPORTANT ARCHITECTURE RULE

FitoutOS = planning + resources + programme
LLD = site operations only

LLD should NOT parse:

quotes
programmes
drawings
schedules

Those belong in FitoutOS.

LLD should only consume:

tasks
project info
programme summary
resource flags
risks
dates

via API later.
